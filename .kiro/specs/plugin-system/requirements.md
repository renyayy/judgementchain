# Nomos プラグインシステム 要件定義書

## 概要

Nomos エディタに **Obsidian ライクなプラグイン機構**を追加する。プラグインは **vault 配下**に配置されたローカルコードとして管理し、Nomos 側が提供する **安全な API** 経由で機能拡張を行う。

## 要件

### 要件1: プラグイン配置とメタデータ

**ユーザーストーリー:** ユーザーとして、vault 内にプラグインを置くだけで Nomos に機能拡張を追加したい。

**受け入れ基準:**

1. WHEN Nomos が起動すると、`<vault>/.nomos/plugins/*/manifest.json` を探索してプラグイン一覧を取得する
2. WHEN `manifest.json` が存在するプラグインは、最低限 `id` / `name` / `version` / `main` を解決できる必要がある
3. WHEN `manifest.json` が不正、または `main.js` が存在しない場合、そのプラグインはスキップし、アプリはクラッシュしない

### 要件2: プラグインの有効化・無効化

**ユーザーストーリー:** ユーザーとして、プラグインを簡単に ON/OFF したい。

**受け入れ基準:**

1. WHEN プラグイン設定画面で有効化/無効化が切り替えられた場合、次回ロード（または即時リロード）で反映される
2. WHEN 無効化されたプラグインの `main.js` は実行されない

### 要件3: エディタ拡張（CodeMirror Extension）

**ユーザーストーリー:** ユーザーとして、エディタの補完・装飾・キーバインド等をプラグインで拡張したい。

**受け入れ基準:**

1. WHEN プラグインが `registerEditorExtension()` を呼び出した場合、その `Extension` はエディタの `extensions` に追加される
2. WHEN 複数プラグインが拡張を登録した場合、すべての拡張が反映される
3. WHEN プラグインが拡張登録中に例外を投げた場合、そのプラグインのみ無効扱いにせず、他は保持してアプリは継続動作する

### 要件4: 右サイドパネル拡張

**ユーザーストーリー:** ユーザーとして、プラグインが右側に独自UIを表示できるようにしたい。

**受け入れ基準:**

1. WHEN プラグインが `registerPanel()` を呼び出した場合、右パネルの切り替えUIにボタンが追加される
2. WHEN パネルが選択された場合、そのプラグインの `render(container)` が描画される
3. WHEN パネル描画中に例外が発生した場合、そのパネルのみエラー表示し、他パネルには影響しない

### 要件5: コマンド拡張

**ユーザーストーリー:** ユーザーとして、プラグインの機能をコマンドとして実行したい。

**受け入れ基準:**

1. WHEN プラグインが `registerCommand()` を呼び出した場合、コマンド一覧（コマンドパレット）に表示される
2. WHEN コマンドを実行した場合、プラグインが登録した `callback` が呼び出される
3. WHEN コマンド実行中に例外が発生した場合、該当コマンドのみ失敗として扱いアプリは継続する

### 要件6: ステータスバー拡張

**ユーザーストーリー:** ユーザーとして、プラグインがエディタ下部に情報を表示できるようにしたい。

**受け入れ基準:**

1. WHEN プラグインが `registerStatusBarItem()` を呼び出した場合、ステータスバーにアイテムが追加される
2. WHEN プラグイン描画中に例外が発生した場合、ステータスバー全体が壊れない

### 要件7: 安全性（ローカルファースト）

**ユーザーストーリー:** ユーザーとして、プラグインが外部送信や危険な操作をしないようにしたい。

**受け入れ基準:**

1. **プラグインが利用できる API は読み取り中心**とし、直接の書き込みや任意の OS コマンド実行を提供しない
2. WHEN プラグインでエラーが発生した場合、アプリが安全に継続動作する

## 非機能要件

### パフォーマンス

1. プラグインロード失敗や 1 プラグインの例外が、通常のエディタ操作に影響しない

### UX

1. プラグインの ON/OFF は視認でき、切り替え時に反映される
